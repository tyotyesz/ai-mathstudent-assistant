# ADR 0002 - Infrastructure as Code

## Context
Infrastructure is minimal (FastAPI + Postgres). Full IaC with Terraform is optional but desired for clarity. However, sprint scope emphasizes a runnable demo.

## Decision
Document intended IaC (Terraform) but ship with Docker Compose as the primary local/demo provisioner. Provide variable-driven configuration in `.env.example` for portability.

## Consequences
- Pros: Quick start, low cognitive load.
- Cons: No cloud automation included; future Terraform needed for production.
- Follow-up: Add Terraform stack (network, DB, container app) later; keep Compose as parity baseline.
